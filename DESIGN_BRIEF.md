# სამეზობლოს აპლიაკცია — UI/UX დავალება

> **მიზანი:** მობილური აპლიკაციის ვიზუალური და ფუნქციური სპეციფიკაცია, რომლის საფუძველზეც შექმნი პროდუქტს.  
> **რეფერენსები:** [Archi Mobile App](https://www.palindroma.com/work/archi-mobile-app) · [DwellUP](https://dwellup.io/)

---

## 1. პროექტის აღწერა

**სამეზობლოს** არის მობილური აპლიკაცია საცხოვრებელი კომპლექსის/ბინადრებისთვის. ერთ ადგილას აერთიანებს:

- საცხოვრებელი ინფორმაცია და დოკუმენტები  
- გადახდები და ხარჯები  
- სერვისის მოთხოვნები (შეკეთება, დახმარება)  
- სიახლეები და განცხადებები  
- მარკეტინგი (აქციები, შეთავაზებები, სიახლეები, ღონისძიებები)

**რეფერენსებიდან რას ვიღებთ:**

| Archi | DwellUP | სამეზობლოს |
|-------|---------|------------|
| ხარისხიანი, საიმედო UI | მეზობლებთან კავშირი | ორივეს საუკეთესო |
| ხელშეკრულება, გადახდები | ხარჯების გადახდა | გადახდების ცენტრი |
| პროექტის განახლებები | სიახლეები, შეტყობინებები | ფიდი + push |
| რეზიდენტის სერვისები | სერვისის მოთხოვნა | მოთხოვნის ფორმა + სტატუსი |
| მასშტაბი (42k+ მომხმარებელი) | CRM მენეჯერებისთვის | MVP: მომხმარებელი, v2: ადმინი |

## 5. მომხმარებლის ნაკადები (User Flows)

### 5.1 პირველი გაშვება
```
შესვლა (მომხმარებელი + პაროლი) → მთავარი (Dashboard)
→ (ოფც.) Onboarding / OTP / ბინის არჩევა — პირველი რეგისტრაციისას
```

### 5.2 გადახდა
```
Tab „გადახდები“ → ბარათი (ბალანსი) → აირჩიე ვალდებულება → დეტალი → „გადახდა“ → წარმატება
```

### 5.3 სერვისის მოთხოვნა
```
Tab „სერვისი“ → + ახალი → კატეგორია → აღწერა + ფოტო → გაგზავნა → სტატუსი (ახალი → მუშავდება → დასრულებული)
```

### 5.4 სიახლე
```
Dashboard → სიახლის ბარათი → დეტალი → (ოფც.) შეტყობინება მეზობლებთან
```

## 6. ვიზუალური სისტემა (Design System)

### 6.1 ფერები

| სახელი | HEX | გამოყენება |
|--------|-----|------------|
| Primary (ტილი) | `#0D9488` | ბრენდი, ტაბი, ლინკები |
| Accent (ნარინჯი) | `#EA580C` | მთავარი CTA (გადახდა) |
| Violet | `#7C3AED` | ღონისძიებები |
| Background | `#FAF8F5` | თბილი ფონი |
| Primary Soft | `#CCFBF1` | ჩიპები, ბარათის ფონი |
| Surface | `#FFFFFF` | ბარათები |
| Text Primary | `#1A1A1A` | სათაურები |
| Text Secondary | `#6B7280` | ქვესათაური, ლეიბლები |
| Success | `#059669` | გადახდილი, დასრულებული |
| Warning | `#D97706` | მოლოდინში |
| Error | `#DC2626` | ვადაგადაცილებული |
| Border | `#E5E7EB` | გამყოფები |

*Archi-ს სტილი: მყარი, პრემიუმ მწვანე. DwellUP: ღია, თეთრი ფონი. კომბინაცია: მუქი primary + ღია ფონი.*

### 6.2 ტიპოგრაფია

| სტილი | ზომა | წონა | გამოყენება |
|-------|------|------|------------|
| H1 | 28px | 700 | ეკრანის სათაური |
| H2 | 22px | 600 | სექცია |
| H3 | 18px | 600 | ბარათის სათაური |
| Body | 16px | 400 | ტექსტი |
| Caption | 13px | 400 | თარიღი, ლეიბლი |
| Button | 16px | 600 | CTA |

**ფონტი (რეკომენდაცია):**  
- iOS/Android: სისტემური (SF Pro / Roboto)  
- ან: **Noto Sans Georgian** — ქართული ტექსტისთვის

### 6.3 spacing & რადიუსი

- Base unit: **8px**
- Padding ეკრანზე: **16px** (ჰორიზონტალური)
- ბარათის padding: **16px**
- Border radius: ბარათი **12px**, ღილაკი **10px**, input **8px**
- Tab bar სიმაღლე: **56px** + safe area

### 6.4 კომპონენტები

| კომპონენტი | აღწერა |
|------------|--------|
| **Primary Button** | სრული სიგანე, Primary ფონი, თეთრი ტექსტი, 48px სიმაღლე |
| **Secondary Button** | outline Primary, თეთრი ფონი |
| **Card** | თეთრი, shadow `0 2px 8px rgba(0,0,0,0.06)`, radius 12px |
| **Quick Action** | 64×64 აიკონი + ლეიბლი, ჰორიზონტალური scroll |
| **Status Badge** | pill: „ახალი“ ყავისფერი, „მუშავდება“ ლურჯი, „დასრულებული“ მწვანე |
| **Bottom Tab** | 5 ტაბი, აიკონი + ლეიბლი, აქტიური Primary |
| **Input** | 48px, border #E5E7EB, focus Primary |
| **List Item** | აიკონი მარცხნით, სათაური + caption, chevron მარჯვნით |
